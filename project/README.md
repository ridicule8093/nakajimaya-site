# 株式会社中島屋 Design System

愛知県弥富市の木製パレット・木材資材BtoB企業、株式会社中島屋のコーポレートサイト向けデザインシステム。テンプレート感を排し、取引先（物流会社・製造業・輸出梱包業者）から見て違和感のない「実在するBtoB企業」のたたずまいを作るための、トーン・色・余白・写真の使い方・セクション設計の参考案。

最終実装は GitHub 上の HTML/CSS を ChatGPT / Codex で編集して行う。このプロジェクトは "完成実装ではなく方向性のたたき台" として読まれることを前提にしている。

ソースリポジトリを直接見られる場合は、現状の実装と本デザインシステムを併読すると改善方針が掴みやすい。

## CONTENT FUNDAMENTALS — 文章とトーン

現行の `CONTENT.md` / `SITE_SPEC.md` を読み込んだ上での観察と原則。

### トーン

- **抑制された敬体**。「〜します」「〜ください」「〜について確認します」が基本形。「〜いたします」もあるが控えめ。
- **大きな主語を使わない**。「業界トップ」「最高品質」「全国対応」のような根拠のない表現は禁止（PROJECT_HANDOFF.md に明記）。
- **対応 "可否" を前提にした言い回し**。「対応します」と断言せず「条件を確認のうえ対応します」「ご相談ください」「対応可否を検討します」。これがBtoBの中小企業として信用できる書き方。
- **熱処理・証明書類は "条件確認前提"**。「熱処理対応します」ではなく「熱処理が必要な場合も、内容を確認のうえ対応します」。

### 一人称・二人称

- 一人称は「中島屋」または「弊社」相当の省略形が中心。「私たち」「我々」は使わない。
- 二人称は「お客様」をあまり使わず、行動主体として「ご相談ください」「お問い合わせください」と命令形敬体で受ける。

### 漢字・送り仮名

- 商品名は **読みやすい漢字＋カタカナ**。「木製パレット」「ワンウェイパレット」「ベニヤ」「合板」。
- 「致します」より「いたします」（ひらがな）。「下さい」より「ください」。これは現状CONTENT.mdに従う。
- 中黒（・）でカテゴリ列挙：「角材・板材」「合板・ベニヤ」「丸太杭・丸太」。

### キャッチコピーの型

良いキャッチコピー例（実在の文章から）：
- 「木製パレット・木材資材の製作、販売に対応」← 抽象でなく実務
- 「寸法・数量・納品条件に応じてご相談ください」← 行動を促す具体性
- 「用途、寸法、数量、納品条件などを確認のうえ、対応可否や見積についてご案内いたします」← BtoBの正しい身構え

避けるべきパターン（このサイトでは絶対に書かない）：
- ❌「未来を支える木材のプロフェッショナル」
- ❌「最高品質をお届けします」
- ❌「お客様に寄り添う」
- ❌「私たちのミッション」
- ❌ "Wood. Logistics. Future." のような英文ポエム

### Eyebrow（小見出しラベル）の英語

`Products`, `Service`, `Strength`, `Flow`, `Contact`, `Company`, `Profile` — 短く名詞で、ALL CAPS／トラッキング広めで使用。これは既存の慣習で、装飾上の信号として有効。**ただし本文に長い英語キャッチを置かない** — 英語はあくまでラベル止まり。

### 絵文字・記号

- **絵文字は使わない**。BtoBでは違和感の元。
- 記号は「・」「：」「／」のみ。「★」「→」「✓」も使わない。本文中で「→」を使いたい時は「から〜まで」の言葉で書く。
- 箇条書きの先頭装飾は CSS で（小さい四角／緑の丸など）。文字記号は使わない。

## VISUAL FOUNDATIONS — 視覚

### 色

詳細は [`colors_and_type.css`](colors_and_type.css)。要約すると：

- **ベース**: 生成り（#fbfaf6）／オフホワイト（#f5f1e9）／白（#ffffff）。純白は使わず、わずかに紙の色を持つ。
- **文字**: 墨色 `#1f1c19`（ほぼ黒、わずかに茶を含む）／補助 `#5c5650` のウォームグレー。
- **罫線**: `#ddd6ca` の温かいベージュグレー。
- **アクセント**: 木材を連想する濃茶 `#6b4a2f`／焼き色 `#4c3422`。「お問い合わせ」ボタンと eyebrow に使う。
- **セカンダリアクセント**: 落ち着いた深緑 `#2f4a3b`。CTAバンドと番号バッジに使う。茶と深緑の2色構成が、木材＋林業＋地場産業の連想を作る。
- **NG**: 蛍光色、青紫グラデーション、淡いパステル多用、純黒、純白。

### タイポグラフィ

見出しは明朝、本文はゴシックの2建て。

- **Body / UI**: Noto Sans JP（GoogleFonts）→ Hiragino Sans → Yu Gothic → Meiryo → sans-serif
- **見出し**: Noto Serif JP（明朝系・老舗の落ち着き）
- **数字**: 見出し以外で目立たせたいときは、英数のみ Inter で別ファミリ化（任意）

詳細とサイズスケールは `colors_and_type.css`。

> 注: 現行コードは system font stack（Hiragino Sans, Yu Gothic, Meiryo）。Web フォントを使うと文字組が安定し AI テンプレ感が減るが、ファイル増・初回ロード遅延のトレードオフがある。**3 案ともに先に system フォントで実装し、必要に応じて Noto を追加** する想定。

### 余白とレイアウト

- **コンテナ幅**: `min(1120px, calc(100% - 40px))` を踏襲（現行の値）。狭くしない。
- **セクション縦余白**: PC 72px / SP 48px。広すぎず（カフェ風になる）、詰めすぎず。
- **垂直リズム**: H2 と本文の間は 16-24px、段落間は 1em（line-height 1.8 と組み合わせて呼吸を作る）。
- **ヒーロー写真**: 高さ 360-440px。フルブリードにはせず、グリッド内に収めて「写真の上に文字を載せない」を基本にする — テキストと写真の対応関係を崩さないため。
- **横並びの上限**: 主要セクションは 2 カラム（テキスト＋写真）か、3-4 カードまで。5 カードはやや詰まり感が出るので 4 までを推奨。

### 角丸

- 現行 CSS は 14-22px の大きめ丸角。これは「カフェ風」「スタートアップ風」寄り。
- 本デザインシステムでは **4-10px に統一**。カード = 6px、ボタン = 4px（または角ピル状の場合のみ 999px）、画像 = 8px。
- 表組とフォーム入力は 4px、または角丸なし。

### ボーダー・影

- **ボーダー優先**。`1px solid #ddd6ca` をカード境界に使う。
- 影は **強くしない**。`0 1px 0 rgba(0,0,0,.04)` か、画像のみ `0 4px 12px rgba(40,30,20,.08)`。
- 現行 CSS の `0 14px 34px rgba(...)` は強すぎ — トーンダウン。
- 「カードに左ボーダー色違い1本だけ」のスタイルは NG（AI テンプレ典型）。

### 背景

- 単色のオフホワイト／淡ベージュの **互い違い** で section を区切る。グラデーションは使わない。
- 例外: ヒーロー部分のみ、現行の `linear-gradient(135deg, #fbfaf6, #efe8dc)` のような **極めて薄い** ベージュグラデは可。視覚的に "紙の表紙" の印象を作る。

### 写真

最重要要素。実写真がデザインを救う。

- **必ず実写真**。AI生成・フリー素材は使わない（PROJECT_HANDOFF.md 明記）。
- 既に9枚の実写真がインポート済み。色温度は **暖色寄り＋自然光**。グレーディングは加えない。
- 写真と説明文の **物理的な対応** を保つ（依頼の重要事項）。同じセクション内に "丸太杭の写真＋角材の説明" のような混在を作らない。
- ヒーロー写真にコピーをオーバーレイしない。隣接配置にして読みやすさを優先。
- 角丸 8px、影は控えめ。

### アニメーション・インタラクション

- **アニメーションはほぼ不要**（依頼に明記）。
- ホバー: ボタンは `--accent` から `--accent-dark` に色変化、所要 150ms。
- リンク: 下線で示すか、ホバー時に色を `--accent` に。
- フェード・スライドイン等の登場アニメは入れない（テンプレ感の元）。
- 唯一許容: ヘッダーの sticky + 微妙な背景透過。

### 透明度・blur

- ヘッダー sticky 時に `backdrop-filter: blur(10px)` + 半透明背景は **許容**（現行を踏襲）。
- それ以外で blur や transparency は使わない。ガラスモーフィズム禁止。

### ボタン

詳細は `colors_and_type.css` と UI kit。

- **Primary**: 濃茶背景＋白文字。形は **角ピル** か **角丸 4px** の2系統。デザインシステムは両方提供して Tweaks で切替。
- **Secondary**: 透明背景＋濃茶ボーダー＋濃茶文字。
- 押下: 100ms で 2-3px だけ暗くなる。サイズは変えない。
- **影は付けない**（現行は重い影あり — トーンダウン）。

### アイコン

詳細は [ICONOGRAPHY](#iconography---アイコン) セクション参照。基本は使わない方針。

## ICONOGRAPHY — アイコン

- **アイコンはなるべく使わない**。BtoB の堅い情報サイトでアイコンが浮くより、無いほうが信用される。
- 例外的に使うのは以下：
  - 番号バッジ（01, 02, 03, 04）— フローセクションで CSS だけで描く（緑の丸＋白数字）。これはアイコンというより番号。
  - 電話／メールの表記 — 「TEL」「Email」のテキスト表記で代用。電話アイコンや封筒アイコンは使わない。
  - 箇条書き先頭の丸／四角 — CSS `::before` で 8×8 の小さな図形。装飾レベル。
- **絵文字は使わない**（前述）。
- **Unicode 記号** は `・` `／` `：` のみ。`→ ★ ✓ ✕` 等は使わない。
- リポジトリ内にアイコンファイル（SVG / ICO / アイコンフォント）は存在しない。新規に icon set を入れる必要も無い — 上記方針で十分。
- ロゴはタイポグラフィのみ：`株式会社中島屋` を 700 weight で表記（現行通り）。シンボルマークは保有していないため作らない。

## 使い方

1. UI kit（`ui_kits/direction_a_traditional/`）を見て、使いたい要素を選ぶ。
2. 採用要素を ChatGPT / Codex に渡して、`ridicule8093/nakajimaya-site` の HTML/CSS を修正してもらう。
3. このシステム内の `colors_and_type.css` を実装側の `:root` に流し込めば、本体トークンの差し替えだけで見た目を合わせられる。
4. 写真は既にインポート済みのものを優先使用。新規撮影が入る場合は色温度・構図を既存と揃える。
