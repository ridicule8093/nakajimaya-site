# 中島屋Webサイト制作 引き継ぎメモ

## 目的

株式会社中島屋の会社案内サイトを、月額制作サービスや高額外注に頼らず、ChatGPT/Codex/GitHubを使って静的HTMLサイトとして制作する。

大きな集客サイトではなく、まずは取引先、紹介先、社名検索で見に来た相手に対して、会社の実在感、事業内容、取扱商品、問い合わせ先が分かる状態を作る。

## これまでの判断

- STUDIOは触ってみたが、自力制作にはやや難しいため一旦主ルートから外した。
- Jimdo AIやNTTタウンページ系も検討したが、正式運用には継続費がかかる。
- 中島屋サイトは内容を頻繁に変えない想定のため、静的HTMLサイトをAIで作成し、GitHub PagesまたはCloudflare Pagesで公開する方針にした。
- GitHubリポジトリ `ridicule8093/nakajimaya-site` を作成し、ChatGPTからファイル作成・更新できる状態にした。

## 写真素材の優先順位

最低限、以下の写真を入れると会社サイトとしての実在感が出る。

1. 木製パレットの積み上げ写真
2. 木材、部材、資材置き場の写真
3. 会社外観または作業場の写真
4. トラック、納品前資材、物流を連想できる写真
5. ベニヤ、合板、丸太杭などの商品写真

AI生成画像や汎用素材より、実際の写真を優先する。

## 今後の推奨作業順

1. GitHub PagesまたはCloudflare Pagesで一度公開する
2. スマホとPCで表示確認する
3. 実写真を最低3枚入れる
4. 電話番号とメールアドレスを確定して `company.html` に反映する
5. 問い合わせフォームを使うか判断する
6. 独自ドメインを決める
7. Google Search Consoleに登録する
8. Googleビジネスプロフィールの情報と整合させる

## 注意事項

- 実態不明の実績、取引先名、認証、保証、許認可、品質規格は勝手に追加しない。
- 「業界トップ」「最高品質」「全国対応」など根拠のない表現は使わない。
- 熱処理や証明書類については、対応可否を条件確認前提で表現する。
- 会社サイトとしては、派手さより信用確認と問い合わせしやすさを優先する。
